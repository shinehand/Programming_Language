# 이진 탐색 트리 (Binary Search Tree)

## 개념

이진 탐색 트리(BST)는 각 노드의 **왼쪽 서브트리**에는 해당 노드보다 **작은 값**이, **오른쪽 서브트리**에는 **큰 값**이 오는 이진 트리입니다.

### 구조

```
        5
       / \
      3   7
     / \ / \
    1  4 6  9
```

## 순회 방법

| 순회 | 설명 | 결과 (위 트리 기준) |
|---|---|---|
| 중위 순회 (In-order) | 왼쪽 → 루트 → 오른쪽 | 1 3 4 5 6 7 9 (정렬) |
| 전위 순회 (Pre-order) | 루트 → 왼쪽 → 오른쪽 | 5 3 1 4 7 6 9 |
| 후위 순회 (Post-order) | 왼쪽 → 오른쪽 → 루트 | 1 4 3 6 9 7 5 |
| 레벨 순서 (Level-order) | BFS 방식 | 5 3 7 1 4 6 9 |

## 주요 연산

| 메서드 | 설명 | 평균 시간 복잡도 |
|---|---|---|
| `insert(val)` | 값 삽입 | O(log n) |
| `search(val)` | 값 탐색 | O(log n) |
| `remove(val)` | 값 삭제 | O(log n) |
| `print_inorder()` | 중위 순회 출력 | O(n) |
| `print_preorder()` | 전위 순회 출력 | O(n) |
| `print_postorder()` | 후위 순회 출력 | O(n) |
| `print_levelorder()` | 레벨 순서 출력 | O(n) |

> ⚠️ 편향 트리(skewed tree)의 경우 최악 시간 복잡도는 O(n)

## 삭제 케이스

1. **자식이 없는 경우**: 그냥 삭제
2. **자식이 하나인 경우**: 해당 자식으로 대체
3. **자식이 둘인 경우**: 오른쪽 서브트리의 최솟값(후계자)으로 대체
